# Дизайн игры "Суд" — court game

**Дата:** 2026-03-05
**Статус:** Утверждён

---

## Обзор

Telegram-игра для 3 игроков в группе. Один игрок — Прокурор, двое — защита. Подсудимым может быть любой персонаж (реальный человек, литературный герой, животное и т.д.). ИИ-судья (OpenClaw) ведёт заседание, реагирует на аргументы, задаёт уточняющие вопросы и выносит приговор.

---

## Роли

| Роль | Карт в руке | Играет | Видит карты |
|------|-------------|--------|-------------|
| ⚔️ Прокурор | 8 | 4 | Только свои |
| 🛡️ Адвокат | 4 | 2 | Свои + Свидетель |
| 👁️ Свидетель защиты | 4 | 2 | Свои + Адвокат |

Итого сыгранных карт: **8** (4 + 2 + 2). На их основании судья выносит приговор.

## Поток игры

### 1. Запуск
- Команда `/суд` в группе
- Бот (от лица судьи) торжественно объявляет об открытии заседания и объясняет правила: кто такой Прокурор, Адвокат, Свидетель, сколько карт, сколько раундов, как играть карты через DM
- Бот просит назвать **подсудимого** — любой персонаж, свободный текст (не обязательно кто-то из игроков: кот Леопольд, Наполеон, ChatGPT и т.д.)
- Бот просит описать **преступление**

### 2. Выбор ролей
- Бот публикует inline-клавиатуру: `⚔️ Прокурор`, `🛡️ Адвокат`, `👁️ Свидетель защиты`
- Каждая кнопка исчезает после нажатия, бот объявляет: "Юра взял роль Прокурора"
- Когда все роли заняты — заседание начинается

### 3. Генерация карт
- ИИ генерирует 16 карт под конкретное дело:
  - 8 карт Прокурору (обвинительные улики/аргументы)
  - 4 карты Адвокату (защитные аргументы)
  - 4 карты Свидетелю (защитные аргументы, отличные от адвокатских)
- Стиль карт: абсурдные, но нарративно связные — из них можно выстроить историю. Не сюрреализм ("машина времени"), а goofy реализм ("видели его у холодильника в 3 ночи")
- Бот присылает карты каждому игроку в **личку (DM)**
- Адвокат и Свидетель видят карты друг друга (для координации стратегии)

### 4. Раунды (4 раунда)
В каждом раунде:
1. Прокурор выбирает и играет 1 карту через DM → бот объявляет её в группе
2. Защита (Адвокат + Свидетель) решают между собой, кто отвечает и какой картой. Каждый имеет лимит 2 карты на игру — стратегия в том, когда их использовать
3. Игрок защиты играет карту через DM → бот объявляет в группе
4. ИИ-судья коротко реагирует на каждую сыгранную карту ("Суд принял к сведению...", "Весьма спорное утверждение...") и при необходимости задаёт уточняющий вопрос

### 5. Финальный приговор
- После раунда 4 судья получает полный протокол заседания (8 сыгранных карт + все реакции)
- Выносит приговор в **нескольких сообщениях** для драматизма:
  - Сообщение 1: резюме позиции обвинения
  - Сообщение 2: резюме позиции защиты
  - Сообщение 3: ключевые противоречия и наблюдения
  - Сообщение 4: **ВИНОВЕН / НЕ ВИНОВЕН** + наказание или оправдание

## Характер ИИ-судьи

- **Строгий, но театральный** — давление чувствуется, но не злой
- **Задаёт уточняющие вопросы** если аргумент расплывчатый или слабый
- **Фиксирует противоречия** — если в раунде 1 утверждается одно, а в раунде 3 обратное, судья это замечает
- **Не принимает отмазки** — "я не так сказал" не работает. Всё сказанное идёт в протокол
- **Не может быть передавлен** одной фразой — требует фактических аргументов

### Таблица `court_games`
| Колонка | Тип | Описание |
|---------|-----|----------|
| id | SERIAL PK | |

### Таблица `court_messages`
Все сообщения текущей сессии заседания — чтобы в промпт судье шёл только контекст этой игры, без лишнего.

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | SERIAL PK | |
| game_id | INT FK → court_games.id | |
| role | TEXT | prosecutor / lawyer / witness / judge / system |
| content | TEXT | Текст карты, реакции судьи, системных событий |
| round_number | INT | Раунд (NULL для системных сообщений) |
| created_at | TIMESTAMP | |

## Архитектура кода

Следует паттерну `game_service.py + game_handlers.py`:

```
src/
  handlers/
    court_handlers.py    # Telegram хендлеры: команды, callback, DM
  services/
    court_service.py     # Игровая логика, генерация карт, промпты для ИИ
```

### AI промпты (4 ключевых)

1. **Открытие заседания** — торжественная речь судьи с объяснением правил
2. **Генерация карт** — создать 16 абсурдно-реалистичных карт под конкретное дело
3. **Реакция на карту** — короткая реплика судьи + возможный уточняющий вопрос, опирается на `court_messages` текущей сессии
4. **Финальный приговор** — многосообщный монолог с отсылками к протоколу заседания

---

## Доставка карт (DM)

Карты доставляются через личку бота. Требование: каждый игрок должен хотя бы раз написать боту в личку перед игрой (стандартное ограничение Telegram).

Адвокат и Свидетель в своей личке видят:
- Свои карты (с кнопками "Сыграть")
- Карты партнёра по защите (без кнопок — только для информации)

## Команды

- `/суд` — начать новое заседание (только в группе)
- `/суд_стоп` — досрочно завершить игру (только для участников или админа)
